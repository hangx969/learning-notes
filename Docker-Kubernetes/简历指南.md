# 核心板块
- 个人信息
	- 姓名
	- 电话：188-8888-8888格式，微信同号
	- 邮箱
	- 求职意向
		- 建议写两三个（运维/Devops/Cloud工程师）
	- 到岗时间
	- 期望薪资
- 专业技能
	- 简历最核心部分，HR关注的位置。表达个人的技能图谱。
	- 按照技术方向进行分类：
		- 编程语言
		- 开发能力
		- 框架
		- 数据库
		- 云原生
		- 容器
		- 云计算
		- 证书
	- 写熟练、掌握、了解即可
	- 可以稍微详细：比如了解SQL性能调优，将接口查询耗时从3s优化至100ms内
	- 舍弃一些过时、与岗位毫无相关的技术
- 工作经历
	- PAR法则：P-负责了什么工作，解决了什么问题，A-具体采取了什么行动，使用了什么工具，R：带来了什么效益
	- 职责与成就：使用强动词开头，比如主导、优化、设计等
	- 量化成果
- 项目经验
	- STAR：背景、任务、行动、结果
- 教育背景
	- 按时间倒序，最新的写前面
- 自我评价
	- 可选。要写对公司的价值。
	- 示例：拥有3年云原生体系实践经验，曾成功支撑业务吞吐量的十倍增长，并实现年度基础设置成本优化30%以上。具备2年全栈开发经验，使用Go和Vue架构用3个月时间开发完成K8s多集群管理平台。
