# Claude Code 创始人红杉大会七个判断

## 摘要

Boris Cherny（Claude Code 创始人）在红杉资本大会上分享了对 AI 时代软件行业的 7 个战略判断，涵盖代码价值重估、软件民主化、能力重心转移、SaaS 护城河瓦解、创业机会窗口、MCP 协议定位、以及 Computer Use 的价值。核心观点：AI 正在将软件工程从手工业推向工业化，代码本身不再稀缺，判断力和领域知识成为真正的竞争优势。

## 关键知识点

1. **代码不再稀缺**：工程师的价值从"会不会写代码"转变为"会不会拆任务、验收结果、管理 Agent"——类似工业革命中工匠到流水线管理者的角色转变
2. **古登堡印刷机类比**：AI 对软件的影响是加速版印刷术革命，软件将在几十年内完全民主化，会做软件像会发短信一样自然
3. **判断力 > 编码能力**：AI 抹平执行门槛后，产品感觉和领域深度理解成为核心竞争力；未来的 generalist 是跨学科全栈（产品+设计+工程），而非工程内部全栈
4. **SaaS 护城河瓦解**：AI 模型能读懂复杂数据映射和工作流，大幅降低切换成本和工作流锁定——过去靠数据沉淀和流程沉淀建起的护城河正在失效
5. **创业者最好的时代**：小团队用 AI 杠杆远超大公司组织重量，AI 越强，小团队加速度与大公司拖拽力的剪刀差越大
6. **MCP 不会死**：MCP 是 Model Interface Protocol（给模型用的接口），区别于 API（Human Developer Interface，给人类开发者用的接口）；AI 时代所有服务默认要 MCP 化
7. **Computer Use 依然重要**：解决 AI 落地最大痛点——企业中大量没有 API 也没有 MCP 的遗留系统，让 AI 像人一样操作老旧系统

## 值得注意

- Boris 将 API 称为 "Human Developer Interface"、MCP 称为 "Model Interface Protocol"——这个定义比官方"Model Context Protocol"更能揭示 MCP 的本质定位
- Claude Code 团队内部实践：设计师、PM、财务、用户研究都在写代码，验证了"跨学科 generalist"的观点
- 文中提到 Opus 4.7 模型在理解和重建复杂工作流方面的能力，是 SaaS 护城河瓦解的技术基础
- 对大公司"负资产"的分析角度独特：不是没钱没人没意愿，而是过去赚钱的组织肌肉恰好卡在 AI 发挥价值的路上
