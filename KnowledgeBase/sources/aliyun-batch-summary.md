# Aliyun 来源批量摘要

## 元信息
- **原始目录**：`Aliyun/`（含计算/网络/存储/数据库/资源管理 子目录）
- **文档数量**：19 篇
- **领域**：Aliyun
- **摄入日期**：2026-04-17

## 整体概述

本文档集涵盖阿里云（Aliyun）核心云服务的学习笔记，围绕 ACP 云计算工程师认证考试展开，系统梳理了计算（ECS、ESS、SMC）、网络（VPC、SLB、CEN、VPN、WAF、DDoS防护、DNS/CDN）、存储（OSS、云盘/快照/镜像、数据湖）、数据库（RDS、DTS）以及资源管理（Landing Zone）等五大领域。内容侧重产品架构、功能特性、计费模式和运维实操场景，适合云计算从业者备考及日常参考。

## 各文档摘要

### 计算

#### [[Aliyun/计算/ECS|弹性计算服务ECS]]
- 核心内容：阿里云 ECS 弹性计算服务全面介绍，包括虚拟化发展历史（Xen -> KVM -> 神龙架构 -> CIPU）、计费模式（按量/包年包月/抢占式）及实例配件。
- 关键知识点：
  - 费用由实例规格、镜像、云盘、公网带宽、快照五部分构成
  - 虚拟化架构经历 Xen -> KVM -> 神龙（裸金属零损耗）-> CIPU 四代演进
  - 寄居架构与裸金属架构是两种常见虚拟化模式
  - 节省计划相当于 ECS 按量付费的预付会员卡

#### [[Aliyun/计算/主机迁移工具SMC|主机迁移工具SMC]]
- 核心内容：Server Migration Center 支持多平台、多环境的服务器迁移，迁移过程无需停机，采用全量+增量方式。
- 关键知识点：
  - 迁移通过中转实例中转数据，再制作镜像完成
  - 支持跨账号、跨Region、跨AZ迁移以及VMWare无代理迁移
  - 迁移时间 = 数据传输时间 + 镜像制作时间，建议预留30%缓冲
  - 可使用 iPerf 工具测试传输速度

#### [[Aliyun/计算/云盘-快照-镜像|云盘、快照与镜像]]
- 核心内容：阿里云块存储（云盘）体系、快照机制与镜像管理，涵盖云盘类型（ESSD/SSD/本地盘）、三副本机制、扩容操作及快照备份策略。
- 关键知识点：
  - 云盘采用三副本机制分布在不同机架，保障高可用
  - ESSD 多重挂载（NVMe共享盘）可同时挂载至最多16台 ECS 实例
  - 第一份快照为全量，后续均为增量快照
  - 一块云盘只能挂载到同一地域、同一可用区的一台 ECS

#### [[Aliyun/计算/弹性伸缩ESS|弹性伸缩ESS]]
- 核心内容：ESS 自动扩缩容服务，可自动创建/删除 ECS 实例并联动 SLB 和 RDS，支持优先级、均衡分布和成本优化三种伸缩策略。
- 关键知识点：
  - 扩容自动关联 SLB 并将 IP 加入 RDS 白名单，缩容反向操作
  - 手动添加的实例缩容时只移出不删除，自动创建的实例会被释放
  - 包年包月实例在伸缩组中作为基础容量保障，缩容时最后移出
  - 期望实例数机制让伸缩活动更可控，失败时可自动重试

### 网络

#### [[Aliyun/网络/VPC|专有网络VPC]]
- 核心内容：阿里云 VPC 专有网络核心架构，包括 CIDR 规划、路由器/交换机/路由表工作原理、隧道隔离机制。
- 关键知识点：
  - VPC 基于隧道技术实现二层和三层网络隔离，不同 VPC 天然不互通
  - 每个交换机网段的第1个和最后3个 IP 为系统保留地址
  - 路由表控制出站流量下一跳，采用最长前缀匹配规则
  - VPC 互通方式：VPC Peering、云企业网 CEN、VPN 网关

#### [[Aliyun/网络/CEN-TR|云企业网CEN与转发路由器TR]]
- 核心内容：CEN 打通不同地域/账号的 VPC 内网连接，TR 作为地域级核心转发组件，支持统一出口、流量安全互访等企业复杂组网场景。
- 关键知识点：
  - 一个 Region 支持创建一个 TR（转发路由器）
  - 支持同地域组网、跨地域组网、混合云组网（专线/VPN/SAG）
  - 可实现所有 VPC 统一互联网出口，集中管控安全
  - 支持多 CEN 实例下的共享服务 VPC 架构

#### [[Aliyun/网络/负载均衡SLB|负载均衡SLB]]
- 核心内容：阿里云负载均衡产品家族（ALB/NLB/CLB）介绍，涵盖底层架构、出入流量规则、高可用与跨地域容灾。
- 关键知识点：
  - ALB 面向七层（100万QPS），NLB 面向四层（1亿并发），CLB 为传统负载均衡
  - 四层 CLB 基于 LVS+keepalived，七层基于 Tengine（Nginx二次开发）
  - CLB 仅提供被动公网访问能力，ECS 主动出公网需另购 EIP/NAT
  - 跨地域容灾需配合智能 DNS（GTM）实现流量调度

#### [[Aliyun/网络/网关-VPN-专线|网关-VPN-专线]]
- 核心内容：NAT网关（SNAT/DNAT）、公网IP产品选型、VPN网关与高速通道专线的使用场景对比。
- 关键知识点：
  - NAT 网关提供 SNAT（出）和 DNAT（入），支持 VPC/交换机/ECS/自定义网段粒度
  - NAT 与 EIP 核心区别：NAT 支持多台 ECS 共用，EIP 只能绑定一台
  - 私网互联产品：CEN（云企业网）、VPC Peering
  - 连接本地 IDC：VPN 网关、高速通道（运营商专线）、智能接入网关 SAG

#### [[Aliyun/网络/DNS-CDN-SSL|DNS、CDN与SSL证书]]
- 核心内容：阿里云云解析 DNS（全局服务、记录类型、智能解析）、CDN 加速原理、SSL 证书管理，以及全局服务与 VPC 服务的区别。
- 关键知识点：
  - 云解析 DNS 是唯一提供100% SLA 的服务
  - TTL 最快可实现1秒全球生效
  - CDN 将源站资源缓存到全球加速节点，用户就近获取
  - 全局服务部署在 VPC 外，数据在外网/内网流转，无法直接设置安全组

#### [[Aliyun/网络/WAF|Web应用防火墙WAF]]
- 核心内容：WAF 对 Web 业务流量进行识别和防护，支持 CNAME 接入和云产品接入，3.0 版本引入 SDK 插件模式大幅简化接入。
- 关键知识点：
  - WAF 仅支持通过域名方式接入，不支持 IP 直接接入
  - 防护覆盖 SQL注入、XSS、WebShell、CC攻击等 OWASP 常见威胁
  - WAF 3.0 支持 SDK 插件模式嵌入 ALB/MSE/FC 网关，无需额外转发层
  - 3.0 支持批量配置防护规则和防护对象组，2.0 需逐域名配置

#### [[Aliyun/网络/DDoS高防|DDoS防护]]
- 核心内容：DDoS 攻击分类（畸形报文/传输层/DNS/连接型/应用层）、攻击特征识别以及阿里云 DDoS 防护产品体系。
- 关键知识点：
  - 常见攻击类型：SYN Flood、CC攻击、Slowloris慢速攻击、DNS随机子域名攻击
  - DNS 随机子域名攻击可穿透全球缓存，直接耗尽权威 DNS 资源
  - 免费版 DNS 遭攻击时会被拉入黑洞（停止解析）
  - 中招特征：CPU/内存异常、流量突增、大量未知访问、连接断开

#### [[Aliyun/网络/云安全中心-云防火墙|云安全中心与云防火墙]]
- 核心内容：云安全中心（态势感知）提供网络层/主机层/应用层纵深防护；云防火墙为 SaaS 化防火墙，覆盖互联网边界、VPC边界、主机边界。
- 关键知识点：
  - 云安全中心通过 Agent 部署，也支持非阿里云主机，卸载后有24小时冷却期
  - 企业版可满足等保合规需求，旗舰版额外支持容器安全
  - 云防火墙区分南北流量（互联网边界）和东西流量（VPC边界）
  - Agent 离线判定：云端10小时收不到心跳即视为离线

### 存储

#### [[Aliyun/存储/对象存储OSS|对象存储OSS]]
- 核心内容：系统对比块存储/文件存储/对象存储三种存储类型的结构与特点，深入解析 OSS 的扁平化 Key-Value 架构、元数据标签和无限扩展性。
- 关键知识点：
  - 块存储结构为数组，文件存储为二叉树，对象存储为哈希表
  - 对象存储无论存储量多大，查找速度一致（扁平化结构）
  - 对象存储不支持部分修改，只能整体覆盖
  - 适用于互联网图片、视频、海量日志备份等场景

#### [[Aliyun/存储/数据湖-HDFS-POSIX|数据湖-HDFS-POSIX]]
- 核心内容：POSIX 标准接口规范、HDFS 分布式文件系统架构（NameNode/DataNode）、数据湖架构理念及存算分离趋势。
- 关键知识点：
  - HDFS 默认3副本，适合大文件高吞吐场景，不适合海量小文件
  - S3/OSS 不完全兼容 POSIX，强行挂载模拟会导致性能极差
  - 数据湖趋势：存算分离，OSS/S3 负责存储，Spark/K8s 负责计算
  - 数据仓库是 ETL（先清洗再存），数据湖是 ELT（先存再分析）

#### [[Aliyun/存储/跨域共享CORS|跨域共享CORS]]
- 核心内容：CORS 跨域资源共享机制原理、简单请求与预检请求（OPTIONS）流程，以及在对象存储/网关/CDN 上的运维配置实践。
- 关键知识点：
  - 同源策略要求协议、域名、端口完全一致，否则浏览器拦截
  - 复杂请求会先发 OPTIONS 预检请求，增加一次 RTT
  - 运维需在流量入口（存储桶/网关/CDN）配置 Access-Control-Allow-* 响应头
  - 前后端分离架构是 CORS 问题的典型触发场景

### 数据库

#### [[Aliyun/数据库/关系型数据库RDS|关系型数据库RDS]]
- 核心内容：阿里云 RDS 关系型数据库的实例类型（基础版/高可用版/集群版）、规格族、存储类型（ESSD/本地SSD/SSD云盘）。
- 关键知识点：
  - 支持 MySQL、SQL Server、PostgreSQL、MariaDB 四种引擎
  - 高可用版一主一备架构，备节点故障时主节点会短暂只读（全局锁，不超过5秒）
  - 集群版备节点可读，最多一主四备共5个节点
  - ESSD PL3 相比 PL1 最高提升20倍 IOPS、11倍吞吐量

#### [[Aliyun/数据库/数据传输服务DTS|数据传输服务DTS]]
- 核心内容：DTS 提供数据同步、迁移、集成、订阅、加工、校验六大功能，详细对比了迁移与同步的区别及 ETL 特性。
- 关键知识点：
  - 同步适合长期实时双向同步，迁移适合短期单向迁移任务
  - 增量迁移任务不会自动结束，需手动停止
  - 不停机迁移需同时选择结构迁移+全量迁移+增量迁移
  - 支持库表列三级对象名映射和条件过滤等 ETL 特性

### 资源管理

#### [[Aliyun/资源管理/Landing Zone|Landing Zone]]
- 核心内容：企业上云框架 Landing Zone 的顶层设计，涵盖资源规划、财务管理、网络规划、身份权限、安全防护、合规审计、运维管理、自动化八大模块。
- 关键知识点：
  - CAF（云采用框架）分为上云战略、上云准备、应用上云、运营治理四阶段
  - 企业上云动机：Capex 转 Opex，加速资源交付和业务创新
  - 网络先行：先确定 Region，再规划 VPC 互通/隔离，最后确定云上云下互联方式
  - Landing Zone 是企业上云的顶层设计和基础框架

## 交叉主题发现
- **VPC 是核心枢纽**：几乎所有网络、计算、数据库文档都依赖 VPC 作为基础网络环境，VPC 的 CIDR 规划、路由表、安全组贯穿全局。
- **安全防护多层联动**：WAF（七层）、DDoS高防（三/四层）、云防火墙（南北+东西流量）、云安全中心（主机层）形成完整的纵深防御体系。
- **弹性伸缩串联计算与网络**：ESS 自动扩缩容联动 ECS、SLB、RDS 三大服务，体现了云服务间的深度集成设计。
- **迁移工具链完整**：SMC 负责主机迁移，DTS 负责数据库迁移/同步，两者配合可覆盖大部分企业上云迁移场景。
- **存算分离趋势**：从 OSS 对象存储到数据湖架构，均强调存储（OSS/HDFS）与计算（Spark/K8s）分离的云原生理念。
- **Landing Zone 统领全局**：Landing Zone 的八大模块（资源/财务/网络/权限/安全/合规/运维/自动化）恰好对应了其他文档中各产品的具体实现。
