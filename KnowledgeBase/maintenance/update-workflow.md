# 🔄 增量维护流程

> [!info] 本文档说明如何保持 KnowledgeBase 知识编译层的持续更新。

---

## 1. 新增一篇文档后，如何更新知识库

```
新增文档
   │
   ├─ 1. 确认该文档有 YAML frontmatter（title, tags, date）
   │
   ├─ 2. 更新 inventory/repository-inventory.md
   │     └─ 在对应领域表格中新增一行
   │
   ├─ 3. 检查是否需要更新 maps/domain-map.md
   │     └─ 如果是新领域或领域内容结构变化
   │
   ├─ 4. 检查是否需要更新 maps/tool-map.md
   │     └─ 如果涉及新工具或已有工具的新使用方式
   │
   └─ 5. 在文档中添加 [[wikilink]] 指向相关已有文档
```

> [!tip] 最低限度
> 至少完成步骤 1 和 5（frontmatter + 双链），其他步骤可批量处理。

---

## 2. 何时补概念页

满足以下任一条件时，应新建概念页：

- ✅ 某个概念/工具在 ==3 篇以上== 文档中被提及
- ✅ 该概念是某个知识域的核心入口（如某个新的云服务、新的 K8s Operator）
- ✅ 已有的概念页中多次引用但找不到对应页

**不建议建概念页的情况：**
- ❌ 只在 1 篇文档中出现的工具名
- ❌ 临时性的版本号或一次性配置项

---

## 3. 何时更新专题地图

- ✅ 某个专题新增 ==3 篇以上== 相关文档
- ✅ 专题的知识结构发生变化（如 CI/CD 工具从 Jenkins 转向 ArgoCD）
- ✅ 新开辟了一个专题领域（如新增"混沌工程"专题）

---

## 4. 何时更新索引页（INDEX.md）

- ✅ 新增了地图页或分析报告
- ✅ 领域成熟度评级发生变化
- ✅ 推荐阅读路径需要调整
- 📅 建议每季度审查一次

---

## 5. 何时做断链巡检

| 触发条件 | 建议 |
|----------|------|
| 批量新增文档后 | 立即巡检 |
| 删除或重命名文件后 | 立即巡检 |
| 定期维护 | 每月 1 次 |

**巡检方法：**

```bash
# 提取 KnowledgeBase 中所有 wikilink 目标
rg -o '\[\[[^\]|]+' KnowledgeBase/ | sed 's/.*\[\[//' | sort -u > /tmp/links.txt

# 列出所有 md 文件（去掉 .md 后缀）
find . -name "*.md" -not -path "./.obsidian/*" | sed 's|^\./||;s|\.md$||' | sort -u > /tmp/files.txt

# 找出断链
comm -23 /tmp/links.txt /tmp/files.txt
```

## 6. 哪些步骤适合自动化，哪些适合人工确认

| 步骤 | 自动化适合度 | 说明 |
|------|:----------:|------|
| Frontmatter 检查 | ✅ 高 | 可脚本扫描缺失 frontmatter 的文件 |
| 断链检测 | ✅ 高 | 可脚本自动化（见上方命令） |
| inventory 更新 | ⚡ 中 | 可半自动生成文件列表，人工补标题和主题 |
| 概念页创建 | ⚠️ 低 | 需要人工判断概念价值和定义 |
| 专题地图更新 | ⚠️ 低 | 需要人工理解知识结构变化 |
| 分析报告更新 | ⚠️ 低 | 需要人工洞察和判断 |
| 命名规范执行 | ⚡ 中 | 可脚本检测不规范命名，人工决定修复 |

## Claude Code 辅助维护

> [!tip] 使用 Claude Code 进行批量维护
> 可以通过 Claude Code 执行以下维护任务：
>
> 1. **断链巡检**：让 Claude Code 扫描并报告断链
> 2. **批量添加双链**：指定目录，让 Claude Code 分析内容并添加相关 wikilink
> 3. **概念页生成**：提供概念名称，Claude Code 可自动搜索仓库并生成概念页
> 4. **inventory 更新**：让 Claude Code 重新扫描并更新盘点文件
>
> 建议指令示例：
> ```
> 请扫描 KnowledgeBase/ 中的所有断链，并更新 broken-links-report.md
> ```
> ```
> 请为"混沌工程"创建一个概念页到 KnowledgeBase/concepts/
> ```
