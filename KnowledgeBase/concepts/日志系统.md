# 日志系统

## 定义
日志系统是可观测性的核心支柱之一，负责日志的采集、传输、聚合、存储和可视化。本仓库涵盖 EFK（Elasticsearch + Fluentd + Kibana）、Loki + Promtail、Kafka 日志缓冲等方案，从二进制部署到 Helm/Operator 部署均有实践记录。

## 核心要点
- 六种日志采集方案：DaemonSet（节点级）、Sidecar（Pod 级）、直写（应用内）、Logging Operator、Fluent Bit + Fluentd 分层、Kafka 缓冲
- EFK vs Loki：EFK 全文索引能力强但资源消耗大；Loki 仅索引标签、成本低但查询灵活性受限
- 审计日志：K8s API Server 审计日志独立于应用日志，需单独配置采集策略
- 日志管理三要素：采集（Fluentd/Promtail）、存储（ES/Loki）、可视化（Kibana/Grafana）
