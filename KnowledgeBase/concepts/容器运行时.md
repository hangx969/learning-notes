# 容器运行时

## 定义
容器运行时（Container Runtime）是负责在操作系统层面创建和运行容器的底层软件。Kubernetes 通过 CRI（Container Runtime Interface）与容器运行时交互。自 Kubernetes 1.24 起，containerd 成为默认运行时，取代了 Docker Engine 中的 dockershim。

## 核心要点
- K8s 通过 CRI（Container Runtime Interface）与容器运行时交互，1.24 版本后彻底移除 dockershim
- containerd 是当前 K8s 默认运行时，从 Docker Engine 中解耦而来，更轻量
- Docker Engine 仍可用于开发环境的镜像构建和本地测试，但生产 K8s 集群推荐 containerd
- 运行时迁移路径：Docker → containerd，需注意 cgroup driver 对齐和 socket 路径变更
