你有没有这种感觉：读了一本好书，过两个月想起某个观点，但怎么也找不到在哪一页；开完一个重要会议，满脑子是要做的事，但三天后只记得一半；手机备忘录、电脑文档、纸质笔记分散在各处，想找一条信息要翻半天。

  

这不是记性差的问题，而是信息太散、没有一个统一的地方管起来。

  

second-brain这个插件，就是来做这件事的。

  

---

  

## 它是什么

  

second-brain是OpenClaw ClawHub上的一个知识管理插件，名字直译就是"第二大脑"。它把你的各种输入——读书笔记、会议记录、灵感碎片、文章摘录——统一存进一个结构化的本地知识库，然后用AI帮你检索、整理、关联。

  

和普通笔记软件最大的区别是：你不需要手动分类整理，直接把内容扔给它，它自己理解语义，建立关联。下次想找的时候，用自然语言问它，它从知识库里把相关内容都找出来，而不是靠你记得当时存在哪个文件夹里。

  

---

  

## 能解决什么问题

  

**场景一：读书笔记不再流失**

  

读书的时候把觉得有价值的段落直接发给OpenClaw：

  

> "记下来：《思考，快与慢》里说的'锚定效应'——人在做判断时会过度依赖第一个接收到的信息。联系到我们的产品定价，先报高价再打折，比直接报低价效果更好。"

  

second-brain把这条笔记存进知识库，同时记录来源、打上标签、和已有的相关知识建立关联。三个月后你在讨论定价策略，问它"我有没有记过关于定价或心理的内容"，它直接把这条和其他相关笔记一起找出来。

  

**场景二：会议记录自动整理**

  

会议结束后，把录音转录文字或者粗糙的流水账发给它：

  

> "把这段会议记录整理一下，提取Action Items和关键决策，存进知识库"

  

它整理出结构化的会议纪要，提取每条待办事项和负责人，存档的同时可以关联到相关项目笔记。

  

**场景三：灵感收集不丢失**

  

走路突然冒出一个想法，发给OpenClaw一句话：

  

> "记一条灵感：可以做一个根据用户历史提问自动生成FAQ的工具，自动更新文档"

  

第二天整理产品想法的时候，问它"最近有没有关于自动化文档的灵感"，它会把这条和其他相关的想法汇总给你。

  

**场景四：主题聚合**

  

某天你要写一篇关于"远程工作效率"的文章，问它：

  

> "帮我整理一下知识库里所有和远程工作、效率管理相关的笔记，按主题归类"

  

它把你过去几个月散落在各处的笔记全部聚合出来，按主题整理好，你有了一份完整的素材库，不需要自己回忆和翻找。

  

**场景五：知识连线**

  

积累一段时间之后，second-brain能帮你发现不同笔记之间的关联：

  

> "我最近记录的这条关于'专注力管理'的笔记，和我之前记过的哪些内容有关联？"

  

它找出知识库里语义相关的笔记，帮你建立原来没有想到的连接。

## 怎么装

  
```
clawcli install second-brain
```
  

初始化知识库存储路径（默认存在本地，不上云）：

  
```
clawcli config second-brain --storage-path ~/second-brain
clawcli config second-brain --init
```
  

如果想让知识库自动备份到指定目录，比如你的Dropbox或者OneDrive同步文件夹：

  
```
clawcli config second-brain --backup-path ~/Dropbox/second-brain-backup
```
  

重启服务：

  
```
openclaw restart
```
  

---

  

## 怎么用

  

最简单的用法：直接把内容发给OpenClaw，它识别意图并调用second-brain存储。

  

存笔记：

  
```
记下来：[内容]
```
  

存会议记录：

  
```
整理这段会议记录并存档：[会议内容]
```
  

查询：

  
```
帮我找找知识库里和[主题]相关的内容
```
  

按时间回顾：

  
```
帮我列出这周存进去的所有笔记
```
  

查询某本书的笔记：

  
```
我之前记过《[书名]》里的哪些内容？
```
