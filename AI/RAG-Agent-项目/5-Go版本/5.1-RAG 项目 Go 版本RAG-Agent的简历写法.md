## 智能知识库系统（基于 RAG 架构） 2025 年 06 月 - 2025 年 10 月

厦门 \*\*\*\*\*\*\*\*\* 科技有限公司（Dxxxxxxxxm）｜后台开发工程师

项目介绍：基于 RAG 构建的智能知识库系统，支持文档上传解析、语义检索与智能问答，旨在解决部门内部知识沉淀分散、获取效率低等问题。

技术栈：Gin、GORM、MySQL、Redis、Kafka、MinIO、Elasticsearch、Apache Tika



核心职责：



基于 Go 的 Gin 框架搭建知识库，结合 GORM 实现 ORM 映射与事务管理；

负责构建高可用文件处理流水线，基于 MinIO 实现支持分片上传与断点续传的异步上传能力，使用 Redis BitMap 精确记录上传状态，结合 Kafka 解耦解析与向量化流程，稳定支撑 TB 级文档处理；

设计并开发基于 Elasticsearch 的混合检索引擎，融合语义向量与关键词倒排索引， <u>重构查询逻辑并优化索引结构，将平均检索延迟从 800ms 降至 200ms </u>；

在检索链路中引入多级权限控制机制（RBAC），实现租户隔离与资源级鉴权；

设计结构化 Prompt 构建逻辑，融合文档内容、来源与相似度元数据，为本地部署的大模型提供高质量上下文；

深度参与系统架构评审与问题排查，推动知识库系统顺利上线后，企业内部知识复用效率提升约 60%，获得业务团队一致认可。

## 智能医疗问诊系统（基于 RAG 架构） 2025.03 - 2025.05｜后端开发工程师

项目描述：该平台融合大语言模型、RAG 检索增强生成和知识图谱，向大众提供医疗咨询、智能分诊和个性化预约服务。

技术栈：LangChain4j、RAG、Pinecone、MongoDB、Redis、Spring WebFlux、通义千问、DeepSeek-V3

个人职责：

* 构建混合检索引擎，结合 BM25 与 Dense Retrieval，基于倒数排名融合（RRF）算法+关键词与语义检索结果，A/B 测试确定最优权重比例为 3:7。

* 向量检索使用 text-embedding-v3 模型（1024维），接入 Pinecone 构建高质量医疗知识库，设置相似度阈值 0.8 提升召回精度。

* 搭建用户画像系统，通过用户症状、科室、历史行为等生成多维向量特征，结合协同过滤和医生评分机制实现个性化推荐。

* 针对高频问答基于 Redis 缓存前 20% 热点向量查询结果，降低检索延迟，平均响应时间缩短至 120ms 内。

* 封装智能预约工具类，实现预约、查询、取消等核心逻辑，通过 @Tool 注解定义功能描述与参数规范，支持 LangChain 工具链自动解析。

* 实现跨会话上下文保持机制，基于 MongoDB + MemoryId 实现多用户隔离与记忆持久化。

* 基于 Spring WebFlux 实现流式输出，返回 Flux 实现 token 级响应推送，优化首帧响应时间。

* 接入通义千问与 DeepSeek-V3 多模型 API，使用 LangChain4j 实现统一大模型调用接口，支持提示词分层构建、角色设定与输出格式约束。
