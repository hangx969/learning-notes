* 项目名称：RAG-Agent RAG 知识库

原来我们这里用的是豆包的 embedding，但豆包突然下架了 241505 模型，导致大家在向量的时候直接干懵逼了，报错了，真的非常离谱。如果让字节去做 JDK，那 Java 真活不到现在，一个模型下架，最起码新的模型是要适配旧的向量维度的，豆包新的模型直接把 2048 干掉了，那和 ES 的维度就不一样了，真的离谱。

所以我加急申请了阿里的 embedding 模型，因为阿里明确了向下兼容，维度参数支持 1024、2048 等。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223545074.png)

[<u>登录阿里百炼 </u>](https://bailian.console.aliyun.com/?tab=model#/api-key)，申请 API key，然后复制，随后需要复制到 application.yml 中：

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223547602.png)

[<u>在这里可以查看 </u>](https://bailian.console.aliyun.com/?tab=model#/model-market?capabilities=%5B%22TR%22%5D\&z_type_=%7B%22capabilities%22%3A%22array%22%7D)通用文本向量-v4

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223550410.png)

会免费送一批额度，基本上够我们RAG-Agent使用了，😄等用完了再充值。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223553433.png)

OK，然后把申请的 API key 复制粘贴到 application.yml 中就好了。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223555677.png)

然后重新启动项目，就说明我们搞定了。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223558768.png)
