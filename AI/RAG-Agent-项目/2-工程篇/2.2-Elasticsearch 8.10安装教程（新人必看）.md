### 步骤 3：启动 Elasticsearch

可以直接进入到 bin 目录，然后执行 `./elasticsearch` 启动 ES。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-4f5a44959a4bd1ba3252f57ce62eb923.png>)

默认情况下，ES 默认是自动配置堆大小的，也就是没有设置固定的内存限制，所以 ES 会根据系统可用内存自动分配，我本机有时候能飙到 30 多个 G 的内存。如果你本机没有这么大的内存空间，你可以通过下面的命令运行：

```plaintext
ES_JAVA_OPTS="-Xms5g -Xmx5g" ./bin/elasticsearch
复制代码
```

-Xms 设置初始堆大小，-Xmx 设置最大堆大小，建议将 -Xms 和 -Xmx 设置为相同值，避免堆动态调整的开销。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-ecebe01466c7722a67498d10fe3c2927.png>)

ES 8.10.0 需要 JDK 17 的版本，大家在跑 ES 的时候尽量先配置 JDK17。

可以通过这个连接对比： [<u>https://www.elastic.co/cn/support/matrix#matrix_jvm</u>](https://www.elastic.co/cn/support/matrix#matrix_jvm)

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-906bf1681d6a18b0141be9ccd63e5b82.png>)


如果 ES 没有按照要求加载对应的 JDK 版本，我们可以这样执行：

## ES安全功能解除方法

ES 的安全功能包括：

1. **HTTPS&#x20;**：所有通信默认使用 HTTPS。

2. **身份验证&#x20;**：需要用户名和密码才能访问 Elasticsearch。

3) **证书生成&#x20;**：安装时会自动生成 TLS/SSL 证书。

Elasticsearch 启动时默认启用了 HTTPS（加密通信），如果尝试通过 HTTP（明文通信）访问 Elasticsearch，会导致了以下错误：

```plaintext
received plaintext http traffic on an https channel, closing connection  
复制代码
```

这表明你的客户端（如 curl 或浏览器）发送的是 HTTP 请求，而 Elasticsearch 配置为仅接受 HTTPS 请求

### 方法 1：使用 HTTPS 访问 Elasticsearch

默认情况下，Elasticsearch 会在安装目录下生成一个自签名证书，并启用 HTTPS。你可以通过以下步骤使用 HTTPS 访问 Elasticsearch。

#### 步骤 1：找到生成的证书

在安装 Elasticsearch 时，系统会提示你保存以下信息：

**①、CA 证书路径&#x20;**：通常位于 `elasticsearch-8.10.0/config/certs/http_ca.crt` 。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-3057ad783b56a6721ac8d856d5314a1f.png>)

**②、 用户名和密码&#x20;**：默认用户是 elastic，密码会在启动时生成。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-86ec26ca8c8d201a6e5f9730075bf3d1.png>)

如果忘记了密码，可以通过以下命令重置：

```bash
./bin/elasticsearch-reset-password -u elastic
```

#### 步骤 2：使用 HTTPS 访问

回到解压目录（注意不是 bin 目录），运行以下命令，指定 CA 证书并使用 HTTPS 协议：

```bash
curl --cacert config/certs/http_ca.crt -X GET "https://localhost:9200" -u elast
```

系统会提示输入密码，输入 elastic 用户的密码后即可访问。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-92f760845e39555d0402884b7b7f49c5.png>)

注意⚠️，要记得把 ES 的密码配置到你本地代码的 application.yml 文件，和前面 ES 生成的密码是匹配的。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-2bec60f09847b805988fde623114baff.png>)

### ~~方法 2：禁用 HTTPS 和安全功能（仅限本地测试环境）~~

如果只是在本地开发环境中使用 Elasticsearch，也可以禁用 HTTPS 和身份验证。

#### 步骤 1：修改配置文件

编辑 config/elasticsearch.yml 文件，把这两个配置项修改为 false。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-d207a55a2e729199e447a33b6fb4be28.png>)

修改对应的内容为：

```yaml
xpack.security.enabled: false  
xpack.security.http.ssl.enabled: false 
复制代码
```

#### 步骤 2：重启 Elasticsearch

重新启动 Elasticsearch：

```bash
./bin/elasticsearch  
```

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-63319d7bda706694519a6b10765225e6.png>)

#### 步骤 3：使用 HTTP 访问

现在可以通过 HTTP 协议访问 Elasticsearch：

```sql
curl -X GET "http://localhost:9200"  
复制代码
```

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-6652f0ae742e657e6ca42ac64de4cb2a.png>)

#### 注意事项

* 禁用安全功能仅适用于开发环境，生产环境中不建议这样做。

* 如果你在 Docker 中运行 Elasticsearch，请确保将 xpack.security.enabled 设置为 false。

这时候，还要确保 application.yml 中的 ES 用的是 HTTP 请求。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-804d70a6c3a793c92e72ba2bfac932de.png>)

否则会在启动后端的时候报错 初始化索引失败。

![](<images/✅Elasticsearch 8.10安装教程（新人必看）-ffe62e3b4a30e254762882f532bd9ec5.png>)

## ES 安装 IK 分词器插件

IK 分词器是阿里开源的一个中文分词工具，主要用于全文检索和文本分析。据说最初是由林良益开发的，是 Lucene 和 Elasticsearch 中最常用的中文分词插件之一： [<u>https://github.com/infinilabs/analysis-ik</u>](https://github.com/infinilabs/analysis-ik)

传统的分词工具往往难以准确处理中文的歧义和复杂语境，而IK分词器可以：识别中文词组、处理专有名词、智能切分复杂句子。

历小冰：IK 分词器

### 为什么要安装 IK 分词器呢？

Elasticsearch 默认的分词器（如 standard analyzer）针对英文等西方语言设计，对中文文本处理效果极其有限。这主要是因为英文单词之间有空格，而中文是连续书写的，没有自然分词的边界，标准的分词器会逐字切分，导致语义完全丢失。

比如说 程序员爱编程，默认的分词器会分割为 程 | 序 | 员 | 爱 | 编 | 程，IK 分词器能够切分为 程序员｜爱｜编程。

IK分词器提供了两种主要分词模式：ik\_max\_word，最细粒度的分词模式，会穷尽所有可能的分词组合，适合全文检索；ik\_smart，智能分词模式，最少分词，尽可能保留语义完整的词组，适合精准匹配。

#### <u>下载并安装插件</u>

IK 分词插件的 [<u>GitHub 主页 </u>](https://github.com/infinilabs/analysis-ik?tab=readme-ov-file)里有提供插件的安装方式。

```bash

复制代码
```
