衡量一门产品的流行程度，其实可以通过一种比较取巧的方法，就是去看培训机构会讲什么技术，他们一般都是比较能抓行业趋势的。



那经过我的观察，已经有不少知名的培训机构开始讲 MinIO 了。这就意味着 MinIO 开始越来越受欢迎了。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224811299.png)



我之前看 MinIO 的星标还没这么多，刚上去一看，增长确实有点小快啊！马上 33k 了！



GitHub 地址： <https://github.com/minio/minio>



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224815388.png)



另外，MinIO 可以用来作为云原生应用的主要存储服务，因为云原生应用往往需要更高的吞吐量和更低的延迟，而这些都是 MinIO 的优势。



### 二、安装 MinIO



MinIO 适用于各种平台，包括 Windows、macOS 和Linux，那我自己刚好有一台 CentOS 服务器，所以我们就以 CentOS 为例来安装 MinIO。



如果直接使用 `wget` `https://dl.min.io/server/minio/release/linux-amd64/minio` 去下载安装包的话，速度非常慢。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224817958.png)



我是选择到官网下载安装包，再通过 Tabby 的 SFTP 上传到服务器的。



MinIO 下载地址： <https://min.io/download#/linux>



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224820902.png)



相对 wget，要快得多！



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224824292.png)



执行 `chmod +x minio` 命令添加权限，再执行 `./minio server /home/minioserver` 命令启动 MinIO 服务， 你可以替换 `/home/minioserver` 为你自己的目录，用来存储数据。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224827479.png)



注意，这里会提示 Console 的端口号是动态生成的，也就是只要你重启了服务，端口号就发生变化了。那我们就按照提示设置了一下固定的端口号，并且确保我们的服务器已经放行该端口号。



我这里通过宝塔面板放行了一个 46747 的端口号。如果服务器上没有开启的话，也需要开启一下。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224830653.png)

### 三、使用 MinIO



通过 MinIO 的后台管理面板就可以上传下载文件了，我们就以图片为例吧。



第一步，新建存储桶，点击提示的链接或者右上角的按钮。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224840013.png)



第二步，填写桶名，点击创建按钮。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224843495.png)



第三步，点击 upload，选择文件上传。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224846759.png)



第四步，点击刷新，选中图片，点击分享按钮，就可以访问图片了。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224850010.png)



只不过，此时的分享链接是有时效的。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224853384.png)



要想获取长久链接，我们需要修改存储桶的访问策略，



点击桶的设置按钮：



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224856663.png)



添加访问规则，只读即可：



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224900256.png)



注意前缀设置为 *`.`* ，否则会访问不成功。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224903312.png)



设置成功后，再回到图片的访问链接，复制红色下划线标出的部分。替换 ip 为你的服务器 ip 地址，端口号 9000 同样要记得在服务器/宝塔面板中放开。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224906391.png)



然后就可以在地址栏里访问图片了。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224909763.png)

### 五、总结



这不比 FastDFS 好使得多？关键是，MinIO 是基于Apache V2 license 100% 开放源代码的，可以无限制、自由免费使用。



![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501224923591.png)



MinIO 不仅适合云原生应用，但还特别适合单体应用，对小公司也非常的友好。



如果你的公司不打算上云的话，那么 MinIO 就是很棒的选择；尤其是当公司只是希望搭建一个内部使用的文件存储服务的话，那 MinIO 就是不二的选择，简单易用！



还好管理。这比直接上传到服务器上要得劲的多，因为有管理后端。
