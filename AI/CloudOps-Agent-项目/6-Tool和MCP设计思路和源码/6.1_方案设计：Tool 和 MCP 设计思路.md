# Agent 需要哪些 Tool ？

要明确需要哪些工具，需先锚定每个Agent的核心目标与场景痛点，再对应工具的价值：



## **对话Agent**

**核心需求&#x20;**：智能问答交互

**所需工具&#x20;**`query_internal_docs` ：该工具基于RAG召回技术，能从向量数据库中快速匹配与问题相关的知识片段。支撑知识检索，回答 API鉴权失败怎么办 等问题时，直接定位文档关键片段



## **运维Agent**

**核心需求&#x20;**：自动告警响应、日志/监控联动排查、故障根因分析。

**所需工具&#x20;**：

* `query_prometheus_alerts` ：实时获取告警详情

* `query_log` ：通过腾讯云CLS MCP实现自然语言日志检索（如 服务下线前5分钟错误日志 ）

* `get_current_time` ：实时获取当前时间

* `query_internal_docs` ：补充运维经验（如故障处理步骤），形成完整排查闭环



**总结&#x20;**：四个工具覆盖所有Agent需求，其中 `query_internal_docs` 是通用支撑工具，其他工具聚焦运维Agent的故障排查场景。



# 核心 Tool 设计

## query\_prometheus\_alerts：告警数据精准提取

**功能定位&#x20;**：实时获取 Prometheus 告警信息，为大模型提供故障诊断的第一手数据。

**实现细节：**

* **API 对接&#x20;**：调用 Prometheus 官方告警查询接口 `GET /api/v1/alerts` 。

* **数据提取逻辑&#x20;**：从 JSON 响应中解析关键字段，结构化输出给大模型：

  * `alertname` ：告警名称（如 服务下线 ），来自 `labels.alertname` ；

  * `description` ：告警详情（如 广告微服务下线 ），来自 `annotations.description` ；

  * `activeAt` ：告警触发时间（如 2025-10-29T08:48:42Z ），直接提取 `activeAt` 字段。

**核心优势&#x20;**：实时性强，能快速获取故障诊断的起点数据。



## query\_internal\_docs：知识库精准召回

**功能定位&#x20;**：检索内部文档（如故障处理手册），为所有Agent提供精准的知识片段检索服务。

**实现细节：**

* **文档存储&#x20;**：采用向量数据库存储文档，支持语义相似度检索；

* **调用触发&#x20;**：当大模型判断需要背景知识时（如 服务下线的处理步骤 ），自动调用该工具，传入关键词（如 服务下线 处理步骤 ），返回Top N相关文档片段。

**核心优势&#x20;**：避免大模型 幻觉 ，答案严格基于检索到的事实性内容。



## get\_current\_time：时间感知能力补充

**功能定位&#x20;**：解决大模型 时间失忆 问题，为Agent提供实时时间信息，辅助时间维度的计算（如告警持续时长计算）。

**实现细节：**

* **极简接口&#x20;**：极简接口返回多格式时间（秒/毫秒级时间戳、YYYY-MM-DD HH:MM:SS字符串），Agent自动前置调用需时间参数的场景。

* **调用时机&#x20;**：当工具需要时间参数时（如 查询过去1小时的告警 ），Agent 自动前置调用该工具。

**核心优势&#x20;**：轻量高效，支撑运维Agent的时间校准需求。



# MCP 集成

## query\_log ：自然语言驱动的日志检索

**功能定位&#x20;**： **<span style="color: inherit; background-color: rgba(255,246,122,0.8)">让运维/业务人员用日常语言查询日志，降低技术门槛</span>**

**实现细节：**

* **MCP 对接&#x20;**：集成 腾讯云CLS MCP ，由MCP自动将自然语言转为查询语句，统一返回结构化日志结果

* **调用逻辑&#x20;**：Agent 将用户问题（如 查询服务下线前 5 分钟的错误日志 ）直接转发给 CLS MCP，MCP 自动生成日志查询语句并返回结果，Agent 再将结果整理后提交给大模型。

**核心优势**

* **低门槛&#x20;**：让业务人员用日常语言查日志（如 服务下线前5分钟的错误日志 ），无需学Lucene/SQL语法

* **动态适配&#x20;**：MCP 自动处理不同日志源的语法差异，统一输出格式。



# 典型场景举例：服务下线告警处理流程

以 服务下线 告警为例，展示 Agent 如何协同 Tool 与 MCP 完成任务：

1. **数据采集&#x20;**：调用query\_prometheus\_alerts获取告警详情（名称、触发时间）；

2. **时间校准&#x20;**：通过get\_current\_time计算告警持续时长（如 已持续11分钟 ）；

3) **日志回溯&#x20;**：调用query\_log传入 广告微服务 下线前5分钟 错误日志 ，CLS MCP返回关键日志片段；

4) **知识补充&#x20;**：大模型触发query\_internal\_docs，检索 服务下线 处理步骤 文档，获取标准化操作指南；

5. **决策输出&#x20;**：整合上述数据，生成含日志摘要、处理步骤、历史案例的故障报告。



# 总结

通过标准化Tool接口（实时告警数据、知识检索、时间感知）与灵活MCP集成（自然语言查询日志），让大模型从 被动对话 升级为 主动诊断 ，高效衔接监控数据、内部知识与业务系统，真正实现AI在运维场景的落地价值
