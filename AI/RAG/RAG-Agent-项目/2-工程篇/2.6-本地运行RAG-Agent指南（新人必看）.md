前置环境准备妥当后，接下来就需要从 GitCode 上拉取源码，然后在本地跑起来RAG-Agent了。

## 一、拉取源码

拉取源码我个人比较喜欢的方式是，使用 GitHub 桌面版，图形化操作起来比较丝滑。

GitCode 地址： [<u>https://gitcode.com/qing_gee/PaiSmart</u>](https://gitcode.com/qing_gee/PaiSmart)

代码采用的是邀请审核机制，获取邀请方法请参考： [<u>https://t.zsxq.com/XBc0a</u>](https://t.zsxq.com/XBc0a)

如果你已经在本地安装了 GitHub 桌面版，可以直接点击 file 这里 clone 仓库。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223701231.png)

然后填写前面的 GitHub 地址或者码云地址，选择 URL 这个 tab。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223704912.png)

我本地已经 download 了。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223708043.png)

点击【show in finder】这里可以看到下面的目录仓库。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223711362.png)

前端的 vue 代码在 frontend 目录在，后端的 Java 代码在 src 目录下。

## 二、启动后端

后端代码，建议用 IntelliJ IDEA 打开。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223714350.png)

运行的话，直接点击运行/debug 按钮，或者直接 run main 方法，都可以。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223717403.png)

这里注意，前置环境 MySQL、Redis、ES、MinIO、Kafka 的配置信息都是对应上的。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223721091.png)

如果有遇到任何 sql 相关的错误，很有可能是数据库结构发生了变化。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223724474.png)

这个时候可以拉取最新的代码，并且把本地的 paismart 数据库删除，重新新建一个空的数据库。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223727709.png)

再次运行 SmartPaiApplication，让 sql 重新执行（我们已经让数据文件自动执行了），如果这次没有 sql 相关的报错，像下面这样，一般是可以正常启动的。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223730710.png)

如果在后续上传文件时发现索引创建失败，那么可以执行 `curl -X DELETE '` `http://localhost:9200/knowledge_base'` 删除，重新启动项目，保证索引是最新的，HTTPS 模式下的方法请参考： [<u>https://t.zsxq.com/JTTyV</u>](https://t.zsxq.com/JTTyV)

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223734154.png)

后端启动成功的标志就是 Tomcat started on port 8081。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223737935.png)

## 三、启动前端

前端的话，推荐用 Trae、VSCode 或者 Cursor 打开，我这里用的是 Trae。启动非常简单，切换到 frontend 目录， <u>然后先执行 </u>`pnpm i` <u>安装依赖 </u>，然后执行 `pnpm run dev` 就可以启动前端了。

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223741135.png)

前端启动成功后，会自动在浏览器打开。项目登录账户密码：admin admin123

![](https://raw.githubusercontent.com/hangx969/upload-images-md/main/20260501223744370.png)
