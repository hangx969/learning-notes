# Awesome DESIGN.md

> 仓库：[VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md)

## 什么是 DESIGN.md

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是 Google Stitch 提出的概念——一个纯文本的设计系统文档，AI Agent 读取后即可生成风格一致的 UI。

- 就是一个 Markdown 文件，无需 Figma 导出、JSON Schema 或特殊工具
- 放到项目根目录，任何 AI 编码 Agent 或 Google Stitch 就能理解你的 UI 应该长什么样

| 文件 | 谁读 | 定义什么 |
|------|------|---------|
| `AGENTS.md` | Coding Agent | 怎么构建项目 |
| `DESIGN.md` | Design Agent | 项目应该看起来什么样 |

## 使用方法

1. 从下面的品牌目录中选一个，把 `DESIGN.md` 复制到你的项目根目录
2. 告诉 AI Agent："build me a page that looks like this"

## 每个 DESIGN.md 包含什么

| 章节 | 内容 |
|------|------|
| Visual Theme & Atmosphere | 氛围、密度、设计哲学 |
| Color Palette & Roles | 语义色名 + HEX + 功能角色 |
| Typography Rules | 字体族、完整层级表 |
| Component Stylings | 按钮、卡片、输入框、导航（含状态） |
| Layout Principles | 间距尺度、网格、留白哲学 |
| Depth & Elevation | 阴影系统、层级 |
| Do's and Don'ts | 设计护栏和反模式 |
| Responsive Behavior | 断点、触摸目标、折叠策略 |
| Agent Prompt Guide | 快速配色参考、即用提示词 |

每个品牌目录包含：`DESIGN.md`（设计系统）+ `preview.html`（视觉预览）+ `preview-dark.html`（暗色预览）
